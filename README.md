# Mille Sabords — Optimal Strategy Solver & Web App

*Mille Sabords!* is a [push-your-luck dice game](game_rules.md) by **Haim Shafir**, distributed by **[Gigamic](https://www.gigamic.com)**. This project is an independent fan creation: a dynamic-programming solver that computes the exact optimal strategy for every turn, paired with a web app that lets you play the game and follow the recommended strategy in real time.

**Play it live:** [supitalp.github.io/millesabords](https://supitalp.github.io/millesabords/)

## How it works

The solver models each turn as a [Markov Decision Process (MDP)](https://en.wikipedia.org/wiki/Markov_decision_process),
where every reachable game state is represented as a tuple
`(n_skulls, held_counts, skull_reroll_used)`. It then applies
[value iteration](https://en.wikipedia.org/wiki/Markov_decision_process#Value_iteration)
to solve the MDP exactly, finding the strategy that maximises the expected
score of the current turn. The resulting value function `V(state)` and the
matching policy are exported to compact JSON files (`docs/data/<card>.json`)
that the web app loads on demand.

## Rule adaptations

The solver implements the published rules faithfully with two deliberate simplifications.

### Nine-of-a-kind: a score tier instead of an instant win

The rules say nine identical symbols wins the game immediately ("Pirate's Magic"). We treat it instead as the highest combo tier, worth **8,000 points**, continuing the doubling pattern from 8 identical:

| Identical dice | 3 | 4 | 5 | 6 | 7 | 8 | **9** |
|---|---|---|---|---|---|---|---|
| Points | 100 | 200 | 500 | 1 000 | 2 000 | 4 000 | **8 000** |

This avoids a mathematical discontinuity in the DP (the "instant win" mechanic requires a separate value function) at negligible strategic cost — the probability of reaching nine-of-a-kind is so small (~1/36 to ~1/1296 from any realistic state) that the recommended actions are identical under either rule.

### Treasure Island: implicit freeze instead of explicit dice placement

The Treasure Island card lets players physically move dice onto the card; those dice score even on a bust. We collapse the "on island / on table" distinction into the existing `held` vector: dice you choose not to reroll are implicitly safe and score on bust. A rational player always freezes every die they keep, so the two formulations produce identical decisions and expected values. This keeps the state space ~40× smaller and the UI simpler.

## Insights from the exact solution

Having an exact solver unlocks concrete answers to questions that are hard to reason about intuitively. One example: what is the **expected score** of a turn for each pirate card — i.e., on average, how many points will a perfect player score when that card is drawn? The [full table](analysis/expected_values.md) has all the numbers. A few highlights: the Pirate card (which doubles your score) is by far the strongest, yielding 1 158 pts on average; the Peace card (no swords allowed, BGA variant) sits just below the no-card baseline at 521 pts; the Pirate Ship card (requires 4 swords to score anything) is the only card with a **negative** expected value under optimal play (−156 pts), meaning a perfect player still expects to bust more often than not.

## Project structure

```
solver/          Python DP solver (model, scoring, actions, DP, stats, report)
docs/            Static web app (Vue 3, no build step)
  app.js           All logic: solver port, Vue components, UI
  index.html       HTML shell
  style.css        Styles
  data/            Precomputed JSON per card (generated by export_data.py)
export_data.py   Build step: solve all cards and write JSON into docs/data/
prebuild.py      Optional: pre-populate the solver disk cache (speeds up export_data.py)
analysis/        Exploratory scripts (stopping policy, guardian analysis, simulations, plots)
tests/           pytest suite
```

## Running locally

**Prerequisites:** [uv](https://docs.astral.sh/uv/getting-started/installation/)

```bash
git clone https://github.com/supitalp/millesabords.git
cd millesabords

# Install dependencies and the project itself
uv sync

# (Optional) pre-populate solver cache — makes export_data.py faster on subsequent runs
uv run python prebuild.py

# Solve all cards and export JSON data for the web app
uv run python export_data.py

# Run the test suite
uv run pytest

# Serve the web app locally
uv run python -m http.server 8080 --directory docs
```

### Analysis scripts

```bash
# Simulate turns and compare DP-optimal vs heuristic strategies
uv run python analysis/compare_strategies.py [--card CARD] [--n N] [--all]

# Trace individual turns under the optimal policy
uv run python analysis/simulate.py [--card CARD] [--n N] [--seed SEED] [--quiet]

# Monte-Carlo verification of the DP solver's expected values
uv run python analysis/verify_solver.py [--card CARD] [--n N] [--all]

# Plot score distributions for all cards
uv run python analysis/plot_cards.py [--n N] [--seed SEED]
```
